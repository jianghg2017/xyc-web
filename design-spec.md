## 1. 设计概述

### 1.1 设计理念
- **专业可信**: 体现公司专业形象，建立客户信任
- **简洁高效**: 信息层次清晰，用户快速找到所需内容
- **现代时尚**: 采用现代化设计语言，展现公司创新能力
- **响应式优先**: 确保全设备一致体验

### 1.2 设计原则
1. **视觉层次清晰**: 通过大小、颜色、间距建立明确的信息层级
2. **一致性**: 全站保持统一的视觉语言和交互模式
3. **可访问性**: 符合 WCAG 2.1 AA 标准，色盲友好
4. **性能优化**: 设计考虑加载性能，图片懒加载，渐进增强

### 1.3 技术栈
- **框架**: React + TypeScript
- **样式**: TailwindCSS
- **图标**: Heroicons / Font Awesome
- **动画**: Framer Motion
- **字体**: 思源黑体 / Noto Sans SC

## 2. 核心页面设计说明

### 2.1 首页 (Home)

#### 设计目标
快速传达公司价值主张，引导用户探索核心内容，建立第一印象。

#### 视觉风格
- **Hero Banner**: 全屏背景图/视频，深色遮罩，白色文字，高对比度
- **核心优势区**: 卡片式布局，图标 + 标题 + 描述，悬停动效
- **产品展示**: 网格布局，产品卡片带阴影和悬停放大效果
- **新闻动态**: 时间线或卡片列表，日期标签突出

#### 关键元素
- **CTA 按钮**: 主色调，圆角 8px，悬停加深 10%
- **导航栏**: 透明背景，滚动后变为白色带阴影
- **合作伙伴**: Logo 墙，灰度显示，悬停恢复彩色

#### 交互要点
- Banner 自动轮播 (5 秒)，支持手动切换和暂停
- 滚动触发动画 (Fade-in, Slide-up)
- 导航栏滚动时固定顶部

---

### 2.2 关于我们 (About Us)

#### 设计目标
展示公司实力、历史和文化，建立品牌认同感。

#### 视觉风格
- **公司简介**: 左图右文或右图左文布局，大图展示团队/办公环境
- **发展历程**: 垂直时间轴设计，节点带图标和年份标签
- **团队介绍**: 圆形头像卡片，悬停显示详细信息
- **资质荣誉**: 证书网格展示，点击灯箱放大

#### 关键元素
- **时间轴**: 左侧时间线，右侧内容卡片，交替布局
- **数据展示**: 大号数字 + 标签，计数动画效果
- **价值观卡片**: 图标 + 标题 + 描述，三列布局

#### 交互要点
- 时间轴滚动时节点依次高亮
- 数字滚动动画 (0 → 目标值)
- 证书图片点击灯箱效果

---

### 2.3 产品中心 (Products)

#### 设计目标
清晰展示产品矩阵，帮助用户快速找到合适产品。

#### 视觉风格
- **筛选栏**: 顶部固定或吸顶，分类标签 + 搜索框
- **产品列表**: 卡片网格 (3-4 列)，统一高度，封面图突出
- **产品卡片**: 图片 + 分类标签 + 标题 + 简短描述 + CTA
- **产品详情**: 大图轮播 + 信息分区块展示

#### 关键元素
- **分类标签**: 胶囊样式，选中态主色背景
- **搜索框**: 圆角输入框，右侧搜索图标，支持回车搜索
- **产品卡片**: 悬停阴影加深，图片轻微放大
- **分页**: 数字分页器，支持跳转

#### 交互要点
- 分类筛选即时刷新列表 (无页面跳转)
- 搜索支持实时建议 (debounce 300ms)
- 产品卡片点击进入详情页
- 支持排序 (最新、热门、价格)

---

### 2.4 解决方案 (Solutions)

#### 设计目标
展示公司解决行业问题的能力，通过案例增强说服力。

#### 视觉风格
- **方案列表**: 与产品类似，但强调行业标签
- **方案详情**: 结构化布局，痛点→方案→优势的视觉流
- **案例展示**: 大图 + 数据成果，对比图展示效果
- **咨询入口**: 固定悬浮或页面底部 CTA

#### 关键元素
- **行业标签**: 彩色徽章，不同行业不同颜色
- **痛点分析**: 图标 + 问题描述，红色警示色
- **方案优势**: 绿色对勾图标，列表展示
- **案例数据**: 大号数字 + 百分比，强调成果

#### 交互要点
- 行业筛选支持多选
- 方案详情锚点导航 (痛点/方案/优势/案例)
- 咨询表单悬浮按钮，随时可见
- 案例图片轮播展示

---

### 2.5 新闻资讯 (News)

#### 设计目标
展示公司动态和行业资讯，提升 SEO 和用户粘性。

#### 视觉风格
- **新闻列表**: 卡片或列表布局，封面图 + 标题 + 摘要 + 日期
- **新闻详情**: 文章布局，大标题，作者信息，正文排版优化
- **分类导航**: 顶部 Tab 或侧边栏筛选
- **相关文章**: 底部推荐，3 列卡片

#### 关键元素
- **新闻卡片**: 日期标签突出，分类徽章
- **封面图**: 16:9 比例，统一尺寸
- **摘要**: 2-3 行，超出省略
- **分享按钮**: 社交媒体图标，悬浮展示

#### 交互要点
- 分类切换平滑过渡
- 新闻详情支持分享 (复制链接/社交媒体)
- 相关文章推荐
- 支持 RSS 订阅入口

---

### 2.6 联系我们 (Contact)

#### 设计目标
提供多种联系渠道，降低沟通门槛。

#### 视觉风格
- **联系信息**: 图标 + 文字，卡片式展示
- **联系表单**: 简洁表单，分步或单页，实时验证
- **地图展示**: 嵌入式地图，标记公司位置
- **社交媒体**: 图标墙，悬停放大

#### 关键元素
- **联系卡片**: 电话/邮箱/地址/工作时间分开展示
- **表单字段**: 姓名、邮箱、电话、公司、咨询类型、内容
- **地图**: 高德/百度地图嵌入，支持点击导航
- **社交图标**: 微信、微博、LinkedIn 等

#### 交互要点
- 表单实时验证 (邮箱格式、必填项)
- 提交成功动画 + 确认消息
- 地图支持缩放和路径规划
- 悬浮联系按钮始终可见

## 4. 配色方案

### 4.1 主色调

```
主蓝色 (Primary Blue)
├── 主色：#1890FF (品牌主色，按钮、链接、高亮)
├── 浅 1:  #E6F7FF (背景、悬停)
├── 浅 2:  #BAE7FF (边框、分隔)
├── 深 1:  #096DD9 (悬停、激活)
└── 深 2:  #0050B3 (强调、深色模式)
```

### 4.2 辅助色

```
成功色 (Success Green)
├── 主色：#52C41A
├── 浅：  #F6FFED
└── 深：  #389E0D

警告色 (Warning Orange)
├── 主色：#FAAD14
├── 浅：  #FFF7E6
└── 深：  #D48806

错误色 (Error Red)
├── 主色：#FF4D4F
├── 浅：  #FFF1F0
└── 深：  #CF1322

信息色 (Info Cyan)
├── 主色：#13C2C2
├── 浅：  #E6FFFB
└── 深：  #0E8A8A
```

### 4.3 文字颜色

```
主要文字
├── 标题：  #1F1F1F (95% 黑)
├── 正文：  #4F4F4F (60% 黑)
├── 次要：  #8C8C8C (40% 黑)
└── 禁用：  #BFBFBF (25% 黑)

链接文字
├── 默认：  #1890FF
├── 悬停：  #40A9FF
└── 访问：  #0958D9
```

### 4.4 背景色

```
页面背景
├── 主背景：#FFFFFF
├── 次背景：#FAFAFA (区块区分)
└── 深色背景：#1F1F1F (页脚、Hero)

卡片背景
├── 卡片：  #FFFFFF
├── 卡片悬停：#FFFFFF (阴影加深)
└── 卡片边框：#F0F0F0
```

### 4.5 配色使用规范

| 场景 | 颜色 | 说明 |
|------|------|------|
| 主按钮 | 主蓝色 | 主要 CTA |
| 次按钮 | 白色 + 蓝色边框 | 次要操作 |
| 链接 | 主蓝色 | 内链、外链 |
| 成功状态 | 成功绿 | 成功提示、完成 |
| 错误状态 | 错误红 | 错误提示、验证失败 |
| 警告提示 | 警告橙 | 注意、提醒 |
| 标题文字 | 主要文字 | H1-H6 |
| 正文文字 | 正文文字 | 段落内容 |
| 辅助文字 | 次要文字 | 说明、标签 |

## 5. 组件设计规范

### 5.1 按钮 (Buttons)

#### 尺寸规范
```

├── 高度：48px
├── 内边距：0 24px
├── 字体：16px, 600
└── 圆角：8px

中按钮 (Medium) - 默认
├── 高度：40px
├── 内边距：0 20px
├── 字体：14px, 500
└── 圆角：8px

├── 高度：32px
├── 内边距：0 16px
├── 字体：14px, 500
└── 圆角：6px
```

#### 样式变体
```
主按钮 (Primary)
├── 背景：#1890FF
├── 文字：#FFFFFF
├── 悬停：#40A9FF
└── 激活：#0958D9

次按钮 (Secondary)
├── 背景：#FFFFFF
├── 边框：#1890FF
├── 文字：#1890FF
├── 悬停背景：#E6F7FF
└── 悬停文字：#0958D9

文字按钮 (Text)
├── 背景：透明
├── 文字：#1890FF
├── 悬停背景：#E6F7FF
└── 无边框

├── 背景：#FF4D4F
├── 文字：#FFFFFF
└── 悬停：#FF7875
```

#### 状态
```
默认 (Default)
└── 正常显示

悬停 (Hover)
├── 背景加深 10%
└── 阴影：0 2px 8px rgba(24, 144, 255, 0.2)

激活 (Active)
├── 背景加深 20%

禁用 (Disabled)
├── 背景：#F5F5F5
├── 文字：#BFBFBF
└── 光标：not-allowed

加载中 (Loading)
└── 显示加载图标，禁用点击
```

## 6. 交互流程说明

### 6.1 页面加载流程

```
用户访问 URL
    ↓
显示加载状态 (骨架屏/Logo)
    ↓
加载首屏内容
    ↓
显示首屏内容
    ↓
后台加载剩余内容
    ↓
完全可交互
```

### 6.2 导航交互流程

```
用户点击导航项
    ↓
高亮当前项
    ↓
页面路由切换
    ↓
滚动到顶部
    ↓
加载新页面内容
    ↓
显示新页面
```

### 6.3 表单提交流程

```
用户填写表单
    ↓
实时验证 (失焦时)
    ↓
点击提交按钮
    ↓
前端验证所有字段
    ↓
验证失败 → 显示错误提示
    ↓
验证通过 → 显示加载状态
    ↓
发送请求到服务器
    ↓
成功 → 显示成功提示，清空表单
    ↓
失败 → 显示错误提示，保留数据
```

### 6.4 产品筛选交互流程

```
用户访问产品中心
    ↓
显示所有产品 (默认)
    ↓
用户选择分类/输入搜索
    ↓
显示筛选结果计数
    ↓
刷新产品列表 (平滑过渡)
    ↓
无结果 → 显示空状态提示
```

### 6.5 轮播交互流程

```
页面加载
    ↓
显示第一张 Banner
    ↓
5 秒后自动切换
    ↓
淡入淡出过渡 (0.5s)
    ↓
用户可点击指示器切换
    ↓
用户可左右滑动 (移动端)
    ↓
鼠标悬停时暂停自动切换
```

### 6.6 滚动动画流程

```
页面滚动
    ↓
检测元素进入视口
    ↓
添加动画类 (fade-in-up 等)
    ↓
执行 CSS 动画
    ↓
动画完成后移除类
    ↓
不再重复触发
```

### 6.7 移动端菜单交互流程

```
用户点击菜单按钮
    ↓
显示遮罩层 (淡入)
    ↓
菜单从右侧滑入
    ↓
禁止背景滚动
    ↓
用户点击菜单项
    ↓
关闭菜单 (反向动画)
    ↓
跳转到对应页面
    ↓
恢复背景滚动
```

## 7. 响应式设计规范

### 7.1 断点定义

```css
/* TailwindCSS 默认断点 */
xs:  480px   /* 小手机 */
sm:  640px   /* 大手机 */
md:  768px   /* 平板 */
lg:  1024px  /* 小笔记本 */
xl:  1280px  /* 桌面 */
2xl: 1536px  /* 大桌面 */
```

### 7.2 响应式策略

#### 导航栏
```
< 768px (移动端)
├── 汉堡菜单按钮
├── 抽屉式全屏菜单
└── 垂直列表

≥ 768px (平板及以上)
├── 水平导航菜单
├── 显示所有菜单项
└── 悬停展开子菜单
```

#### 产品/方案列表
```
< 640px (手机)
└── 单列布局

640px - 1023px (平板)
└── 双列网格

≥ 1024px (桌面)
└── 三列或四列网格
```

#### 页脚
```
< 768px (移动端)
└── 单列堆叠布局

≥ 768px (平板及以上)
└── 三列或四列布局
```

### 7.3 字体响应式

```css
/* 标题字体 */
h1: clamp(24px, 5vw, 48px)   /* 32-48px */
h2: clamp(20px, 4vw, 36px)   /* 28-36px */
h3: clamp(18px, 3vw, 28px)   /* 24-28px */
h4: clamp(16px, 2.5vw, 22px) /* 20-22px */

/* 正文字体 */
body: 14px (移动端) / 16px (桌面)

/* 辅助文字 */
small: 12px (全局统一)
```

### 7.4 间距响应式

```css
/* 区块内边距 */
section: 
  - 移动端：16px / 24px
  - 桌面端：48px / 64px

/* 卡片内边距 */
card:
  - 移动端：16px
  - 桌面端：24px

/* 元素间距 */
gap:
  - 小：8px / 12px
  - 中：16px / 24px
  - 大：32px / 48px
```

### 7.5 图片响应式

```css
/* 图片尺寸 */
.cover-image:
  - 移动端：100% 宽度，高度自适应
  - 桌面端：固定比例 (16:9 或 21:9)

  - 统一容器尺寸
  - object-fit: cover

  - 圆形，固定尺寸

  - 桌面端：64px
```
